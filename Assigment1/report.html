My Report

My design decisions

The overall colour scheme of my website is a white background with a grey menu bar, grey headings with
black underlined subheadings. All of the text is in san-sarif. Every image that doubles as an link has a description.
At the bottom of the page there is a home button, an arrow that is a return to the top of the page button.
Each car being reviewed has an image of the car with a brief description and dot points of its main features.

Why it is a good design

The design of the website is built with purpose in mind. The content of the website displays the information of the websites,
and it is created to be clear and understandable. The grey navigation menu and headings contrasts well with the white which allows the content to be seen.
The black text also contrasts well with the white background.

The design conforms to web standards, this includes using CSS to implement style. The CSS is in separate files to the HTML and the HTML documents
contain the content of the website with CSS styling in the HTML when required. The website design is quick and responsive, loading content quickly and changing
to different pages quickly.

The design is focused around on the user and the usability of the website. This includes the effectiveness of the website in its ability to allow the user
to view the information on the website. The efficiency allowing the user to simply navigate the website and the satisfaction of the design and its performance

The design has common characteristics to other similar websites, this includes car review websites like Drive.com, carreview.com and whiccar.com.
This allows the design to conform to web conventions, specifically to the design conventions of other car review websites. This also aids in the ease
to remember the website layout as the layout of the website will seem familiar to
the users as it will be similar to how other websites are displayed and how the user navigates other websites.

How this design conforms to Web standards

The website conforms to web standards in a few ways. This includes having HTML and CSS separate and having different CSS files for different device types.
The website has also been built to be responsive.

This website is accessible, cross device and cross browser compatible. The design allows users to access the website on desktop, tablet or mobile.
This website also allows users to utilise different browsers with the content still being viewable.

This website also meets web conventions this includes having a banner at the top of the page with the logo on the top left with sectional navigation,
the logo at the top is made to be a home button, back to top link at the bottom of the page, and a site map available at the bottom of the page.

How this design conforms to Accessibility standards

The website conforms to accessibility standards including the WCAG 2 and WC3 Design principles.
The website is perceivable as it has a simple design through the use of a white background and black text.
But the information is not lost with the simple design. The website is operable on a range of devices and has clear navigation,
this is to allow people with a disability to still use the website and not restrict their accessibility. The website is understandable as
text content is readable and the website is operable in a simple and predictable way, the website uses context information and each element on the page is clear.
The websites is also able to be navigated without a keyboard or mouse e.g. via touchscreen. The website is also robust as it is compatible with older browsers
or non-standard browsers.

How users will find it easy to navigate on any device

The users will find it easy to navigate the website on any device as the site includes a menu bar at the top and a simple footer bar with a home button,
and a back to top button to make it easier for users to get to the top and the logo is a link to the home page. A more specific example of easy navigation
would be on the home page that not only includes what is discussed above but it also has a sub menu for user to click on to take them to a specific section of the page.
Each specific car page has a menu at the bottom of the page leading to other EV car pages. This increases the navigation of the webpage as it reduces the need
for users to return to the EV overview page to find the next vehicle they wish to view. The website also has a site map at the bottom of the page to help users
understand the navigation of the website.

Why the department should adopt your design

This design should be adopted by the department as it conforms to good web design principles, it incorporates accessibility,
the design of the website is not overly complicated and can be easily changed if needed, and the site conforms to the “car review” format.
Furthermore if the department wanted to change or add a new electric vehicle to their website they would only need to copy
one of the electric vehicles html page and change the information and images. This is also true for the design of the website,
if the colour or layout needed to be changed the CSS files are designed so it is easy to change the design.

Problems Faced

Problem:

On smaller screens I was unable to get text to break to a new line when the size of the screen changes

Attempted Solutions:

In the CSS file for the p element is tried to set the font size to adjust to vw and percentage.
I also attempted to use other CSS styling methods like word-break but to no avail.

Used solution:

Using the br tag in HTML to break the line to ensure the text would fit on any screen size.


Problem:

When using the Footer attribute in HTML instead of the div called “Bottomnav”, the formatting of the Footer
would break and not conform to the design of the website even when formatting was applied to the Footer in CSS

Attempted Solutions:

- Assigning CSS design to the Footer element in the CSS file
- Assigning style to the HTML elements

Used Solution:

Creating a Div called “Bottomnav” allowed the formatting to be consistent across the site.



Problem:

Sometimes elements would not space out properly or cause issues when using the padding-bottom or padding-top style in CSS.

Used Solution:

I found using empty an p element or h elements and the applying padding-bottom or padding-top styling in CSS to those elements would work.
